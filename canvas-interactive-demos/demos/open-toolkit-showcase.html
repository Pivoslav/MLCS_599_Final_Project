Open toolkit showcase

This page intentionally loads a few libraries from public CDNs (Mermaid, Three.js, Google model-viewer). Other demos in this folder stay CDN-free for picky LMS hosts; use this lab on GitHub Pages or when your course allows external script.

Ideas, not course requirements: STL/3D, diagrams, and archive embeds are spice—pair with SOURCES.md for rights. For 3D prints of real monuments, prefer models you have permission to redistribute or CC-licensed meshes.

1. Mermaid — structure without a build step

Render flowcharts, sequence diagrams, and Gantt charts from plain text in the repo. Students can edit diagrams in GitHub’s web UI; you can export SVG for slides. mermaid.js.org

2. Embedded STL — 3D in the syllabus (and at the makerspace)

ASCII STL is just text; you can version it in GitHub, diff it, and parse it in the browser with Three.js. Same mesh can feed a slicer for a tactile “Bronze Horseman” / Neva flood discussion object—STL is not only for engineering courses.

3. model-viewer — GLB/GLTF (lighter than raw STL for the web)

Google’s <model-viewer> handles AR on some devices. Swap src for your own CC-licensed GLB (e.g. simplified architectural prop). modelviewer.dev

Demo asset is a placeholder astronaut—not thematic; it proves the embed path.

4. Web Audio — one-shot ambient without MP3 files

Oscillator + gain envelope: no external audio file; works offline once the page is loaded. Use sparingly; respect prefers-reduced-motion for auto-play.

5. GitHub-native affordances (no extra hosting)

  • GitHub Pages — you already ship the CYOA; add docs/ or branch builds for student micro-sites.
  • Discussions — Q&A templates per week; link a run summary from the game.
  • Issues + forms — structured bug reports for “which scene / path / browser.”
  • Actions — lint JSON scenes, spell-check narrative, deploy Pages on tag.
  • Gist — paste anonymized run exports for TA review.
  • Shields.io — badges in README (build, license, Pages).
