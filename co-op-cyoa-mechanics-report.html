1. Plurality vote + tie-break + rare override

Inspired by vote-based interactive drama (e.g. As Dusk Falls FAQ — majority wins, random tie-break, special overrides).

Group choice
Votes: A1 · B1 · C1 → tie-break rule fires → B wins (demo)
Code / UI effort
Design risk
Learning payoff

What it would take: State for three votes (or reuse your ballot demo), a resolver function (majority → choice index; tie policy), optional override tokens stored in state. Mostly game-app.js + one modal or footer strip; no new narrative content required.

Why learning: Students experience disagreement under constraint—the same skill as reading factional politics. Why fun: brief drama each beat without role-play improv. Not too complicated: one rule card on screen; hide entirely outside co-op.

Prototype: canvas-interactive-demos/demos/mechanic-co-op-timer-ballot.html

Code / UI effort

What it would take: A rotation index in state; only the clerk’s pointer can click; others see the same vote tallies or prompts. Trivial UI badge: “Clerk this scene: Reform.”

Why learning: Mirrors advice vs signature in bureaucracy and salons. Fun: fair turns at the mouse. Simple: no new narrative; one counter per scene advance.

Rises on risky publish choices; drops on withdraw / calm beats. Threshold unlocks extra censor event (optional).
Code / UI effort
Design risk
Learning payoff

What it would take: One new number in state, hooks on a subset of choices, optional branch check. Talk-only beats = scene flag noAdvance: true with a single “Continue” after discussion.

Why learning: Makes cumulative risk tangible (not just three independent stats). Fun: “we heated the room.” Keep simple: start with heat affecting one winter table row only.

Code / UI effort
Design risk
Learning payoff

What it would take: After certain choices, a “Pass to Reform to answer” button that only highlights their co-op card; Skip this beat logs nothing and advances. Policy text in README for instructors.

Why learning: Supports classroom psychological safety and equal airtime. Fun: low-stakes delegation. Simple: mostly UI affordances, almost no logic.

Why learning: Historians disagree partly because sources differ. Fun: “what did you see?” reveal. Scope control: ship three scenes first, not the whole graph.

6. Predict → reveal

Inspired by Quandary — anticipate reactions before locking a recommendation.

Code / UI effort
Design risk
Learning payoff

What it would take: Before applying choice effects, show three radio predictions (“Who pays most?”); store none in save; on next line, show a one-sentence “model answer” gloss. Pure UI + short strings.

Why learning: Hypothesis testing is core historical thinking. Fun: quick bet, no grading. Simple: optional modal; skip in speed-run mode.
